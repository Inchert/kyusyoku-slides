報告の基本

報連相（ホウレンソウ）の使い分け

種類 定義 タイミング 具体例
報告 指示された業務の結果・進捗を伝える 完了時、中間、異常発生時 「先日ご相談した件、対応完了しました」
連絡 関係者に情報を共有する 知っておくべき情報が発生したとき 「A事業所のチーフが来月退職予定です」
相談 判断に迷うことについて意見を求める 自分の判断だけでは不安なとき 「施設からこういう要望があり、どう対応すべきか相談です」

報告の鉄則

良い報告

  1. 結論から先に

    「結論から申し上げますと〜」

  2. 事実と意見を分ける

    「事実として〜、私の見解としては〜」

  3. 数字を使う

    「前月比+3件、目標比-2%」

  4. 悪いことほど早く

    トラブルは即報告

悪い報告

  1. 経緯から長々と

    「まず背景から説明しますと〜」

  2. 事実と意見が混在

    「〜だと思うので〜だったと思います」

  3. 曖昧な表現

    「だいたい」「おそらく」「たぶん」

  4. 悪いことを後回し

    報告が遅れるほど事態悪化

レポート・報告書の書き方

PREP法

論理的でわかりやすい文章を書くためのフレームワーク

P: Point
結論・要点
→
R: Reason
理由
→
E: Example
具体例・データ
→
P: Point
結論の再確認

PREP法の具体例

報告例：食材費削減の提案

P（結論） 「A事業所の食材費率を改善するため、発注基準の見直しを提案します」
R（理由） 「現在、食材費率が目標を4%超過しており、主な原因は特定食材の廃棄増加です」
E（具体例） 「過去3ヶ月の廃棄記録を分析したところ、野菜類の廃棄が月平均15kgあり、約3万円のロスが発生しています。発注基準を週単位から日単位に変更することで、約60%削減が見込めます」
P（結論） 「よって、来月から発注基準の見直しを実施したいと考えています」

週報・月報のテンプレート

週報の構成

  1. 今週のサマリー（3行以内）
  2. KPI実績
    • 食数実績 vs 計画
    • クレーム件数
    • 欠員状況
  3. トピックス（重要事項3つ程度）
  4. 課題と対応
  5. 来週の予定・重点事項

月報の構成

  1. 今月のサマリー（5行以内）
  2. 収支実績
    • 売上・原価・利益
    • 食材費率・人件費率
  3. 品質指標（クレーム・衛生）
  4. 人員状況（欠員・離職・採用）
  5. 施設との関係
  6. 課題と改善計画
  7. 来月の重点施策

説得の技術

説得が必要な場面

上司への説得

  • 人員増加の要請
  • 設備投資の提案
  • 新しい取り組みの承認

現場への説得

  • 新ルールの導入
  • 業務改善の実施
  • シフト変更の協力依頼

施設への説得

  • 価格改定の交渉
  • サービス変更の提案
  • クレーム対応後の信頼回復

説得の3要素

ロゴス（論理）

データ・事実に基づく
論理的な説明

「数字で示す」

パトス（感情）

相手の感情に訴える
共感を得る

「心を動かす」

エトス（信頼）

日頃の言動による
信頼の積み重ね

「この人の言うことなら」

説得のテクニック

テクニック 説明 具体例
メリットを先に 相手にとっての利点を最初に伝える 「この方法だと作業時間が30分短縮できます」
選択肢を示す 複数の選択肢を提示して選ばせる 「A案とB案があります。A案はコスト重視、B案は品質重視です」
実績を示す 成功事例を提示する 「他の事業所で同じ取り組みを行い、20%改善しました」
リスクも伝える デメリットも正直に伝えることで信頼を得る 「導入初月は一時的に負担が増える可能性があります」
相手の立場で考える 相手が何を重視しているかを理解する 「施設様のご懸念は○○だと思いますので、△△で対応します」

根拠ある説明

「なぜ？」に答えられる説明

根拠のない説明

  • 「なんとなくそう思います」
  • 「経験上、そうです」
  • 「みんなそう言っています」
  • 「以前からそうしています」

根拠のある説明

  • 「過去3ヶ月のデータによると〜」
  • 「他事業所の実績では〜」
  • 「業界標準では〜」
  • 「○○の調査によると〜」

根拠として使えるもの

根拠の種類 具体例 説得力
自社データ 過去の実績、KPI推移、他事業所の事例 高い
業界データ 業界平均、ベンチマーク、統計データ 高い
第三者評価 施設からのフィードバック、監査結果 高い
専門家の見解 栄養士の意見、コンサルの提言 中程度
現場の声 スタッフからのヒアリング結果 補助的

伝わる文章の書き方

文章の基本ルール

1. 一文を短く

悪い例：本日、A事業所を巡回したところ、チーフから人員不足の相談があり、確認したところ早番のパートが2名退職予定であることが判明し、採用活動を開始する必要があると考えます。

良い例：本日、A事業所を巡回しました。チーフから人員不足の相談を受けました。確認したところ、早番のパートが2名退職予定です。採用活動の開始が必要です。

2. 主語と述語を近づける

悪い例：私は、昨日の会議で議論された新しい衛生管理ルールについて、現場への導入を来月から開始することを提案します。

良い例：新しい衛生管理ルールの現場導入を提案します。来月から開始したいと考えています。
